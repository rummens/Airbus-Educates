# Your Workshop Environment

**A five-minute tour of the lab environment before you start — so the tools never get in the way of the learning.**

Every lab in the DCS Academy runs entirely in your browser: a split terminal, a code
editor, a visual cluster view, and a feedback form, all on one screen. This short
orientation lab walks you around each of them and shows you the *clickable actions* the
labs use — you click, you rarely type. Once you've done this once, every later lab feels
familiar.

- **Track:** Core / Fundamentals · Lab 0
- **Audience:** Beginner — no container, Kubernetes, or DCS experience needed
- **Duration:** ~10 min
- **Format:** Hands-on, guided — split terminal, runs in your own OpenShift session namespace
- **Prerequisites:** A DCS login. Nothing else.

## By the end of this lab you'll be able to

- Describe the two halves of the dashboard — instructions and the work area — and the three kinds of clickable action.
- Use the split terminal: run a command in the upper pane and in the lower pane.
- Open a file in the editor from a clickable action.
- Switch to the Console tab (the Kubernetes Dashboard) for a visual view of your namespace.
- Find and submit the feedback form.

## What you'll do

You'll click your way around the session: run a couple of `oc` commands in the two
terminal panes, open a file in the editor, visit the Console tab, and leave feedback.
No concepts yet — just the lay of the land, so the real labs can focus on DCS.
