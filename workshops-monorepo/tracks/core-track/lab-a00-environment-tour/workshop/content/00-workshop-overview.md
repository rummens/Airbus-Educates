Welcome to the **{{< param product_name >}}** Academy. Before you learn anything about
the platform itself, this short lab gives you a tour of the environment every lab runs
in — the terminal, the editor, the cluster view, and the feedback form — so you can focus
on the learning, not on figuring out the tools.

This is the very first lab in the Foundations track. It assumes **no prior container,
Kubernetes, or {{< param product_short >}} knowledge** — just a DCS login.

{{< note >}}
**First time in one of these labs?** You're in the right place — that's exactly what this
lab covers. For a written reference you can keep open, see the
[DCS Academy environment guide]({{< param dcs_docs_base_url >}}/academy/environment-guide).
{{< /note >}}

## What You'll Learn

By the end of this workshop you will be able to:

- Describe the two halves of the dashboard — the **instructions** and the **work area** — and the three kinds of **clickable action**.
- Use the **split terminal** — run a command in the upper pane and in the lower pane.
- Open a file in the **editor** from a clickable action.
- Switch to the **Console** tab (the [Kubernetes Dashboard](https://kubernetes.io/docs/tasks/access-application-cluster/web-ui-dashboard/)) for a visual view of your namespace.
- Find and submit the **feedback** form.

## Prerequisites

None. A DCS login is all you need — every tool is already running in your browser.

## Your Environment

This session is a browser-based environment with an embedded **terminal**, a **code
editor**, and a **Console** tab, all on one screen. Nothing to install and no login to
do — it's ready. All commands are run with [`oc`](https://docs.openshift.com/container-platform/latest/cli_reference/openshift_cli/getting-started-cli.html),
the OpenShift command line, and you already have your own {{< param product_short >}}
project scoped just to you.

## Time and Difficulty

- **Estimated time:** 10 minutes
- **Difficulty:** Beginner
