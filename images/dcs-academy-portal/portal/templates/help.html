Help

How the {{ product }} Academy works

Everything runs in your browser — nothing to install. This guide covers how the catalog is organised, what you see inside a lab, and how to manage your sessions.

The big picture

Tracks, labs and trophies

{{ icon('layers')|safe }}

Tracks

A track is a themed learning path — a group of related labs in a sensible order (for example Foundations, Developer, Security). On the catalog each track is a collapsible section.

{{ icon('rocket')|safe }}

Labs

A lab (or workshop) is a single hands-on exercise with its own private environment. Each shows its difficulty and rough duration. Work through them in order — later labs build on earlier ones.

{{ icon('trophy')|safe }}

Trophies

Finish a lab and you earn its trophy. A lab counts as complete when you submit the short feedback form at the end. Your Trophies page shows what you've earned and what's left in each track.

Your progress is tied to your login, so trophies and the “Continue where you left off” shortcut follow you across visits and devices.

Getting started

From catalog to running lab

1
Pick a lab

Browse or search the Catalog. Use the difficulty filters, or open a track and pick a lab to see its full description.

2
Start session

Click Start session. It opens in a new browser tab so the catalog stays open behind it. A progress screen shows your private environment being provisioned (a workshop pod, and for some labs a virtual cluster).

3
You're dropped into the lab

When it's ready you're taken in automatically. If the progress bar seems stuck for a while, a page refresh resumes right where it left off.

Inside a lab

The session window

The screen is split in two: instructions on the left, your work area on the right. You read a step on the left and carry it out on the right.

Instructions toolbar (top-left)

  • {{ icon('circle-check')|safe }} Home — jump back to the lab's first page.
  • ← Back / → Forward — move one page at a time through the lab.
  • {{ icon('list-checks')|safe }} Contents — the page list; jump straight to any section.
  • Print — a printable view of the instructions.

Session controls (top-right)

  • {{ icon('clock')|safe }} Countdown — time left before the session expires. Sessions are generous but not forever; finish or restart before it runs out.
  • {{ icon('refresh')|safe }} Reload — refresh the workspace if a tab looks stuck.
  • {{ icon('grid')|safe }} Fullscreen — expand the work area to the whole window.
  • {{ icon('logout')|safe }} Leave — return to the portal. The session keeps running; reopen it any time from My sessions.
  • {{ icon('cog')|safe }} Menu — extra options for the session.
The work area

Terminal, Console and Editor

Switch tabs with the headers at the top of the right pane. Which tabs appear depends on the lab; most have all three.

{{ icon('terminal')|safe }} Terminal

A shell in your environment, starting in ~/exercises. Many labs use a split terminal — an upper and a lower one (numbered 1 and 2 in the screenshot above) — so you can, for example, watch something in one while you make a change in the other. Commands use the OpenShift CLI, oc.

{{ icon('monitor')|safe }} Console

A visual web console for your session's namespace — browse Deployments, Pods, Services, ConfigMaps and Secrets without the command line. Labs send you here to see what you just created.

{{ icon('edit')|safe }} Editor

A full VS Code editor open on ~/exercises for viewing and editing files. Use the Explorer on the left to open files; many clickable actions in the instructions open and edit files here for you.

You rarely type

Clickable actions

Wherever the instructions show a coloured box, you can click it instead of typing:

{{ icon('terminal')|safe }}

Run a command

Click a command box and it types and runs in the terminal for you.

{{ icon('edit')|safe }}

Edit a file

Click to open a file in the editor and apply the change — you watch it happen, no hand-editing.

{{ icon('list-checks')|safe }}

Verify

A check box runs a test to confirm the step worked before you move on.

If a box doesn't seem to do anything, make sure the right tab (usually Terminal) is visible, then click again.

Managing your work

The session manager

My sessions (top-right menu) lists every lab you currently have running.

{{ icon('rocket')|safe }} Open
Jump back into a running session — the same environment you left, with your work intact.
{{ icon('refresh')|safe }} Delete
End a session completely and free its resources. This removes the environment, so anything unsaved is lost — but it also lets you start that lab fresh next time instead of resuming a broken or half-finished one. You'll be asked to confirm.

You can run a limited number of sessions at once. If you hit the limit, the portal offers to free one so you can start another.

Tips

Getting the most out of a lab

  • Follow the order. Steps build on each other, and verify checks gate progress on purpose.
  • Watch the visible tab. Running a command switches you to the Terminal — switch to the Console when a step points there.
  • Take your time. Read the why, not just the what.
  • Stuck? Re-read the last expected output, confirm the right tab is visible, and re-run the action.
  • Tell us how it went. The feedback form at the end takes 15 seconds, earns your trophy, and shapes the next labs.
