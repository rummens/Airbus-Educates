Help

How the labs work

First time in a {{ product }} lab? Everything runs in your browser — nothing to install. This page explains what you see and how to get through a lab.

Getting started

From catalog to running lab

1
Pick a workshop

Browse the tracks on the home page and open a workshop for its summary, difficulty and duration.

2
Start session

Click Start session. A provisioning screen shows live progress while your private environment spins up (a workshop pod, and for some labs a virtual cluster).

3
You're dropped into the lab

When ready you're redirected automatically. Your session is bound to your login — come back later and you resume the same one.

The lab screen

Two panes

{{ icon('book')|safe }}

Left — Instructions

Step-by-step content. You scroll through and click the highlighted actions.

{{ icon('grid')|safe }}

Right — Work area

Tabs — Terminal, Editor, Console, Slides — where the work happens. One at a time; click a tab header to switch. Which tabs appear depends on the lab.

You rarely type

Clickable actions

{{ icon('terminal')|safe }}

Run a command

A blue box with a shell command. Click it — it types and runs in the terminal for you (using the OpenShift CLI, oc).

{{ icon('edit')|safe }}

Edit a file

Click to open a file in the editor and apply the change — highlight, insert or replace. You watch it happen; no hand-editing.

{{ icon('list-checks')|safe }}

Verify (examiner)

A green check box runs a test to confirm the step worked (pod running, service reachable) before you move on.

If a box doesn't seem to do anything, make sure the right tab (usually Terminal) is visible, then click again.

The tabs

What each tab is for

{{ icon('terminal')|safe }} Terminal
A shell in your session, starting in ~/exercises. Many labs use a split terminal — an upper and a lower — e.g. one to watch while the other makes a change.
{{ icon('edit')|safe }} Editor
A full VS Code editor on ~/exercises for viewing and editing files.
{{ icon('monitor')|safe }} Console
The cluster web console — a visual view of what you deploy (Deployments, Pods, Services). Labs send you here to see what you created.
{{ icon('book')|safe }} Slides
Concept slides alongside the hands-on steps, when a lab includes them.
Tips

Getting the most out of a lab

  • Follow the order. Steps build on each other; verify checks gate progress on purpose.
  • Watch the visible tab. Running a command switches you to the Terminal — switch back to the Console or app when a step points there.
  • Take your time. Sessions have a generous limit; read the why, not just the what.
  • Stuck? Re-read the last expected output, check the Terminal tab is visible, re-run the action. Diagnose steps include hints.
  • Tell us how it went. At the end, the feedback form takes 15 seconds and shapes the next labs.
